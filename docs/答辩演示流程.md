# 答辩演示流程

本文档用于课程最终答辩演示。建议先运行自动化测试和健康检查，再按下面顺序操作浏览器和 Swagger。

## 启动服务

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python -m pytest -q
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

浏览器打开：

```text
http://127.0.0.1:8000
```

健康检查地址：

```text
http://127.0.0.1:8000/health
```

## 演示脚本

1. 打开 `/` 首页，说明系统用于根据客户数据生成保险产品推荐，当前版本已经形成数据概览、模型训练、模型评估、推荐生成、历史查询和 CSV 导出的完整闭环。
2. 打开 `/data`，展示训练集和评估集基础统计。重点说明训练集 5822 条、评估集 4000 条、字段 86 个，正样本比例约 6%，因此模型评估不能只看 Accuracy。
3. 展示训练集样本比例图，说明该图表来自 `GET /api/charts/dataset`。
4. 打开 `/train` 或 `/docs` 中的 `POST /api/models/train`，依次训练三类模型：

```json
{"model_name": "logistic_regression"}
{"model_name": "decision_tree"}
{"model_name": "random_forest"}
```

5. 说明每次训练都会写入 SQLite 训练记录和指标记录，模型文件保存为 `saved_models/{run_id}.joblib`，同时维护 `saved_models/latest_model.joblib`。
6. 打开 `/evaluate`，展示最近一次模型指标和模型指标对比图。说明对类别不平衡数据需要重点看 Precision、Recall、F1、AUC。
7. 打开 `GET /api/models/compare`，说明该接口返回三种模型各自最近一次训练指标，前端模型指标图使用同一批数据。
8. 打开 `/recommend`，执行 `POST /api/recommend/predict` 生成推荐结果：

```json
{"limit": 20}
```

9. 展示推荐概率、`high` / `medium` / `low` 推荐等级和推荐理由，说明推荐结果已保存到 SQLite。
10. 展示推荐等级分布图，说明该图表来自 `GET /api/charts/recommend-levels`。
11. 打开 `GET /api/recommend/history`，展示最近推荐批次的历史记录；如需复核某次结果，可带 `run_id` 查询。
12. 打开 `GET /api/recommend/export`，下载 `recommendations.csv`，说明该文件用于最终汇报或人工复核推荐名单。
13. 打开 `/docs`，按模块说明最终 API：数据上传、模型训练与对比、推荐历史与导出、三类 ECharts 图表接口。
14. 展示 `docs/screenshots/pytest 测试通过截图.png` 或现场运行 `python -m pytest -q`，说明最终验证结果为 `78 passed`。

## 讲解重点

- 持久化：训练记录、指标记录和推荐结果都写入 SQLite，服务重启后仍可查历史。
- 多模型：逻辑回归、决策树、随机森林使用相同数据流程，便于横向比较。
- 可视化：数据页、评估页、推荐页分别覆盖数据分布、模型指标和推荐等级。
- 推荐闭环：生成推荐结果后，可以查看历史、下载 CSV，并在答辩中展示完整链路。
- 类别不平衡：正样本比例不足 6% 时，Precision、Recall、F1、AUC 比单一 Accuracy 更能说明推荐模型质量。
- 数据隔离：上传数据写入 `data/runtime/`，不会覆盖 `data/raw/` 中的课程样例数据。
- 运行产物：SQLite 数据库、运行时数据和模型文件都属于本地运行产物，不提交到仓库。
