# 答辩演示流程

本文档用于第二阶段现场演示。建议先运行一遍自动化测试，再按下面顺序操作浏览器和 Swagger。

## 启动服务

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python -m pytest -q
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

浏览器打开：

```text
http://127.0.0.1:8000
```

## 演示脚本

1. 打开 `/` 首页，说明系统用于根据客户数据给出保险产品推荐，第二阶段新增 SQLite 持久化、多模型对比、图表展示和推荐结果导出。
2. 打开 `/data`，展示训练集和评估集基础统计，再展示正负样本比例图。说明该图表来自 `GET /api/charts/dataset`。
3. 打开 `/train` 或 `/docs` 中的 `POST /api/models/train`，依次训练三类模型：

```json
{"model_name": "logistic_regression"}
{"model_name": "decision_tree"}
{"model_name": "random_forest"}
```

4. 说明三次训练都会写入 SQLite 训练记录和指标记录，模型文件保存到 `saved_models/latest_model.joblib`。
5. 打开 `/evaluate`，展示最近一次模型指标和模型指标对比图。说明对类别不平衡数据不能只看 Accuracy，还要重点看 Precision、Recall、F1、AUC。
6. 打开 `GET /api/models/compare`，说明该接口返回三种模型各自最近一次训练指标，前端模型指标图使用同一批数据。
7. 打开 `/recommend`，执行 `POST /api/recommend/predict` 生成推荐结果：

```json
{"limit": 20}
```

8. 展示推荐概率、`high` / `medium` / `low` 推荐等级和推荐理由，说明推荐结果已保存到 SQLite。
9. 展示推荐等级分布图，说明该图表来自 `GET /api/charts/recommend-levels`。
10. 打开 `GET /api/recommend/history`，展示最近推荐批次的历史记录；如需复核某次结果，可带 `run_id` 查询。
11. 打开 `GET /api/recommend/export`，下载 `recommendations.csv`，说明该文件用于最终汇报或人工复核推荐名单。
12. 打开 `/docs`，按模块说明 Phase 2 API：模型训练与对比、推荐历史与导出、三类 ECharts 图表接口。

## 讲解重点

- 持久化：训练记录、指标记录和推荐结果都写入 SQLite，服务重启后仍可查历史。
- 多模型：逻辑回归、决策树、随机森林使用相同数据流程，便于横向比较。
- 可视化：数据页、评估页、推荐页分别覆盖数据分布、模型指标和推荐等级。
- 推荐闭环：生成推荐结果后，可以查看历史、下载 CSV，并在答辩中展示完整链路。
- 类别不平衡：正样本比例较低时，Precision、Recall、F1、AUC 比单一 Accuracy 更能说明推荐模型质量。
