# 接口说明

本文档记录系统已实现页面和 API，便于前后端、算法和文档同学对接。当前接口口径对应最终交付版本：上传数据写入 `data/runtime/`，仓库样例数据保留在 `data/raw/`，模型文件按训练批次版本化保存。

## 已实现页面

| 路径 | 方法 | 说明 |
| --- | --- | --- |
| `/` | GET | 系统首页 / 第一阶段 MVP 控制台 |
| `/data` | GET | 数据概览页面，展示训练集和评估集基础统计及正负样本比例图 |
| `/train` | GET | 模型训练页面，展示三类模型训练 API 调用提示，不在 GET 请求中触发训练 |
| `/evaluate` | GET | 模型评估页面，展示最近训练指标和模型指标对比图；无记录时显示空状态 |
| `/recommend` | GET | 推荐预测页面，展示推荐结果说明、推荐等级分布图、历史查询和导出入口 |

## 已实现基础接口

| 路径 | 方法 | 说明 |
| --- | --- | --- |
| `/health` | GET | 系统健康检查 |
| `/docs` | GET | FastAPI 自动生成的接口文档 |

## 已实现 API

| 路径 | 方法 | 说明 |
| --- | --- | --- |
| `/api/datasets/profile` | GET | 读取当前训练集和评估集的基础统计；优先使用 `data/runtime/`，不存在时回退到 `data/raw/` |
| `/api/datasets/upload` | POST | 上传训练集和评估集，写入 `data/runtime/`，并清理已生成模型文件 |
| `/api/models/train` | POST | 训练指定模型并保存 SQLite 训练记录、指标记录和模型文件；`model_name` 支持 `logistic_regression`、`decision_tree`、`random_forest` |
| `/api/models/runs` | GET | 查看最近训练记录列表 |
| `/api/models/compare` | GET | 查看三类模型各自最近一次训练指标，用于模型对比 |
| `/api/models/evaluate` | GET | 查看最近一次模型评估指标；没有训练记录时返回 404 |
| `/api/recommend/predict` | POST | 对评估数据生成推荐概率、推荐等级和推荐理由，并保存推荐结果 |
| `/api/recommend/history` | GET | 查询最近一次或指定 `run_id` 的推荐结果，支持 `limit` 限制返回数量 |
| `/api/recommend/export` | GET | 导出最近一次或指定 `run_id` 的推荐结果 CSV |
| `/api/charts/dataset` | GET | 返回训练集正样本、负样本数量，用于数据概览图 |
| `/api/charts/model-metrics` | GET | 返回模型名称和 Accuracy、Precision、Recall、F1、AUC 数组，用于模型指标对比图 |
| `/api/charts/recommend-levels` | GET | 返回 `high`、`medium`、`low` 推荐等级数量，用于推荐等级分布图 |

## 数据上传接口

```http
POST /api/datasets/upload
Content-Type: multipart/form-data
```

表单字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `train_file` | file | 训练集文件，支持 `.xlsx` 和 `.csv` |
| `eval_file` | file | 评估集文件，支持 `.xlsx` 和 `.csv` |

上传要求：

- 两个文件都必须包含目标字段 `移动房车险数量`。
- 训练集和评估集的特征字段需要保持一致。
- 上传成功后，系统会清理旧模型文件，需要重新训练模型后再生成推荐。
- 未设置上传管理 token 时，该接口可直接调用。
- 如果设置了环境变量 `INSURANCE_RECOMMENDATION_ADMIN_TOKEN`，请求需要携带同值请求头 `X-Admin-Token`。

运行时数据路径：

```text
data/runtime/data.xlsx
data/runtime/eval.xlsx
```

该目录已通过 `.gitignore` 排除，不会提交到仓库。

## 推荐结果字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `customer_id` | string | 客户编号或数据行号 |
| `probability` | float | 模型预测购买概率 |
| `recommend_level` | string | 推荐等级：`high`、`medium`、`low` |
| `reason` | string | 简短推荐理由 |

## 指标说明

本项目数据存在类别不平衡问题。训练集 5822 条，正样本 348 条，正样本比例 5.98%；评估集 4000 条，正样本 238 条，正样本比例 5.95%。模型评估时不能只看 Accuracy，还需要重点关注：

- `Precision`：被推荐客户中实际可能购买的比例。
- `Recall`：实际可能购买客户中被模型识别出来的比例。
- `F1`：Precision 和 Recall 的综合指标。
- `AUC`：模型区分正负样本的整体能力。
