# 接口说明

本文档记录系统第一版计划提供的页面和 API，便于前后端、算法和文档同学对接。

## 已有接口

| 路径 | 方法 | 说明 |
| --- | --- | --- |
| `/` | GET | 系统首页占位信息 |
| `/health` | GET | 系统健康检查 |
| `/docs` | GET | FastAPI 自动生成的接口文档 |

## 计划页面

| 路径 | 说明 | 负责人 |
| --- | --- | --- |
| `/data` | 数据概览页面，展示样本数、字段数、正负样本比例 | 前端 + 数据 |
| `/train` | 模型训练页面，提供训练按钮和训练记录 | 前端 + 后端 + 算法 |
| `/evaluate` | 模型评估页面，展示 Accuracy、Precision、Recall、F1、AUC | 前端 + 算法 |
| `/recommend` | 推荐结果页面，展示客户推荐概率和推荐等级 | 前端 + 后端 + 算法 |

## 计划 API

| 路径 | 方法 | 说明 |
| --- | --- | --- |
| `/api/datasets/profile` | GET | 读取 `data/raw/data.xlsx` 和 `data/raw/eval.xlsx` 的基础统计 |
| `/api/models/train` | POST | 训练模型并保存训练记录 |
| `/api/models/runs` | GET | 查看历史训练记录 |
| `/api/models/evaluate` | GET | 查看模型评估指标 |
| `/api/recommend/predict` | POST | 对输入客户或评估数据生成推荐结果 |
| `/api/recommend/export` | GET | 导出推荐结果 |

## 推荐结果字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `customer_id` | string | 客户编号或数据行号 |
| `probability` | float | 模型预测购买概率 |
| `recommend_level` | string | 推荐等级：`high`、`medium`、`low` |
| `reason` | string | 简短推荐理由 |

## 指标说明

本项目数据存在类别不平衡问题，正样本比例约为 6%。模型评估时不能只看 Accuracy，还需要重点关注：

- `Precision`：被推荐客户中实际可能购买的比例。
- `Recall`：实际可能购买客户中被模型识别出来的比例。
- `F1`：Precision 和 Recall 的综合指标。
- `AUC`：模型区分正负样本的整体能力。
