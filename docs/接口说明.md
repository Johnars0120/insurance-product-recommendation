# 接口说明

本文档记录系统第一版已实现和计划提供的页面、API，便于前后端、算法和文档同学对接。

## 已实现页面

| 路径 | 方法 | 说明 |
| --- | --- | --- |
| `/` | GET | 系统首页 / 第一阶段 MVP 控制台 |
| `/data` | GET | 数据概览页面，展示训练集和评估集基础统计 |
| `/train` | GET | 模型训练页面，展示可用模型和训练 API 调用提示，不在 GET 请求中触发训练 |
| `/evaluate` | GET | 模型评估页面，展示当前进程最近一次训练指标；无记录时显示空状态 |
| `/recommend` | GET | 推荐预测页面，展示预测 API 和结果字段说明，不在 GET 请求中自动预测或训练 |

## 已实现基础接口

| 路径 | 方法 | 说明 |
| --- | --- | --- |
| `/health` | GET | 系统健康检查 |
| `/docs` | GET | FastAPI 自动生成的接口文档 |

## 已实现 API

| 路径 | 方法 | 说明 |
| --- | --- | --- |
| `/api/datasets/profile` | GET | 读取 `data/raw/data.xlsx` 和 `data/raw/eval.xlsx` 的基础统计 |
| `/api/models/train` | POST | 训练逻辑回归模型并保存最近一次训练记录和模型文件 |
| `/api/models/runs` | GET | 查看当前进程内最近一次训练记录列表 |
| `/api/models/evaluate` | GET | 查看最近一次模型评估指标；没有训练记录时返回 404 |
| `/api/recommend/predict` | POST | 对评估数据生成推荐概率、推荐等级和推荐理由 |

## 计划 API

| 路径 | 方法 | 说明 |
| --- | --- | --- |
| `/api/recommend/export` | GET | 导出推荐结果，暂未实现 |

## 推荐结果字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `customer_id` | string | 客户编号或数据行号 |
| `probability` | float | 模型预测购买概率 |
| `recommend_level` | string | 推荐等级：`high`、`medium`、`low` |
| `reason` | string | 简短推荐理由 |

## 指标说明

本项目数据存在类别不平衡问题，正样本比例约为 6%。模型评估时不能只看 Accuracy，还需要重点关注：

- `Precision`：被推荐客户中实际可能购买的比例。
- `Recall`：实际可能购买客户中被模型识别出来的比例。
- `F1`：Precision 和 Recall 的综合指标。
- `AUC`：模型区分正负样本的整体能力。
