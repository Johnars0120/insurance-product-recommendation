# 协同开发说明

本文档用于说明本项目在 GitHub 上的协作方式。组员第一次参与开发时，建议先完整阅读本文件，再开始修改代码。

## 1. 总体原则

本项目采用“一个稳定主分支 + 多个功能分支”的轻量协作方式。

- `main`：稳定版本，只放已经确认能运行的代码。
- `feature/data-model`：数据与算法开发分支。
- `feature/backend`：后端接口开发分支。
- `feature/frontend`：前端页面开发分支。
- `feature/docs`：文档、测试记录、答辩材料分支。

所有组员都应该在自己的功能分支上开发，完成后通过 Pull Request 合并到 `main`。

## 2. 第一次获取项目

命令行方式：

```powershell
git clone https://github.com/Johnars0120/insurance-product-recommendation.git
cd insurance-product-recommendation
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

启动系统：

```powershell
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

访问地址：

```text
http://127.0.0.1:8000
```

如果使用 GitHub Desktop，可以选择 `File -> Clone repository`，选择本仓库后克隆到本地，再用 VS Code 打开项目目录。

## 3. 每次开发前

开始写代码前，先同步最新代码：

```powershell
git switch main
git pull origin main
```

然后切换到自己的功能分支：

```powershell
git switch feature/data-model
```

如果本地还没有这个分支，可以使用：

```powershell
git fetch origin
git switch -c feature/data-model origin/feature/data-model
```

其他同学把分支名替换成自己负责的分支即可。

## 4. 开发完成后

查看修改内容：

```powershell
git status
```

提交代码：

```powershell
git add .
git commit -m "feat: 完成数据概览统计功能"
git push origin feature/data-model
```

提交信息要能说明本次修改内容，不要写“修改”“更新”“111”这类信息。

推荐提交信息格式：

```text
feat: 新增功能
fix: 修复问题
docs: 修改文档
test: 添加或修改测试
style: 调整样式
refactor: 重构代码
```

## 5. Pull Request 合并流程

功能完成后，在 GitHub 页面上创建 Pull Request：

```text
base: main
compare: 自己的 feature 分支
```

Pull Request 中需要说明：

- 本次完成了什么功能。
- 修改了哪些主要文件。
- 如何运行或测试。
- 是否需要其他组员配合。

组长检查后再合并到 `main`。

## 6. 模块接口约定

公共接口、字段名、文件路径不要随意修改。涉及以下内容时，先在群里确认：

- Excel 数据字段名。
- 模型输入输出格式。
- API 路由路径。
- 数据库表字段。
- 页面模板名称。
- 项目目录结构。

第一版推荐结果统一采用以下格式：

```json
{
  "customer_id": "客户编号或行号",
  "probability": 0.82,
  "recommend_level": "high",
  "reason": "模型预测购买概率较高"
}
```

推荐等级规则：

- `high`：推荐概率大于等于 `0.70`
- `medium`：推荐概率大于等于 `0.40` 且小于 `0.70`
- `low`：推荐概率小于 `0.40`

## 7. 每日进度同步

每天晚上在群里同步一次进度，格式固定为：

```text
今天完成：
遇到问题：
明天计划：
需要他人配合：
```

进度同步要具体。例如“完成了 data.xlsx 字段统计，并输出正负样本比例”比“看了数据”更有价值。

## 8. 不要提交的内容

以下内容不要提交到 GitHub：

- `.venv/` 虚拟环境目录。
- `__pycache__/` 缓存目录。
- 本地数据库文件，如 `*.db`、`*.sqlite`。
- 训练生成的模型文件，如 `*.joblib`、`*.pkl`。
- 临时文件和 Office 缓存文件。

原始课程数据可以放在 `data/raw/`，处理后的数据和导出结果不提交。

## 9. 给组员的群通知模板

```text
各位同学，我已经建立了 GitHub 私有仓库，用于统一管理保险产品智能推荐系统的代码、数据和文档。

仓库地址：
https://github.com/Johnars0120/insurance-product-recommendation

请大家先接受 GitHub 邀请，然后克隆仓库，按照 README.md 和 docs/协同开发说明.md 进行环境配置。

协作规则：
1. main 分支只放稳定代码，不直接修改 main。
2. 每个人在自己的功能分支上开发。
3. 开发前先 pull 最新代码。
4. 完成一个小功能后 commit，并写清楚提交信息。
5. 功能完成后发 Pull Request，由组长统一检查合并。
6. 涉及接口、字段、目录结构修改时，先在群里确认。

第一阶段目标：系统能启动、能读取数据、能训练模型、能显示评估指标、能生成推荐结果。

每天晚上请按这个格式同步进度：
今天完成：
遇到问题：
明天计划：
需要他人配合：
```

```text
docs/phase2/README.md

```
