| `app/services/history_service.py` | 后端和推荐成员负责 |
| `data/raw/` | 原始数据保持不变 |

## 图表目标

| 页面 | 图表 | 数据接口 |
| --- | --- | --- |
| `/data` | 训练集正负样本比例饼图 | `/api/charts/dataset` |
| `/evaluate` | 模型指标对比柱状图 | `/api/charts/model-metrics` |
| `/recommend` | 推荐等级分布柱状图 | `/api/charts/recommend-levels` |

## 图表 API 目标

| 接口 | 方法 | 返回内容 |
| --- | --- | --- |
| `/api/charts/dataset` | GET | 正样本、负样本数量 |
| `/api/charts/model-metrics` | GET | 模型名称和各指标数组 |
| `/api/charts/recommend-levels` | GET | high、medium、low 数量 |

## 页面规则

1. GET 页面不能触发训练。
2. GET 页面不能自动预测。
3. 图表通过 `fetch()` 请求 API 获取数据。
4. 图表容器要有固定高度，避免页面抖动。
5. 如果没有训练或推荐记录，页面显示空图或提示，不报错。

## 开发步骤

1. 新增 `app/routers/charts.py`。
2. 在 `app/main.py` 注册 chart router。
3. 在 `base.html` 引入 ECharts。
4. 在 `data.html` 增加 `dataset-chart` 容器。
5. 在 `evaluate.html` 增加 `metrics-chart` 容器。
6. 在 `recommend.html` 增加 `recommend-level-chart` 容器。
7. 在 `main.css` 增加图表容器样式。
8. 补充 API 测试和页面容器测试。
