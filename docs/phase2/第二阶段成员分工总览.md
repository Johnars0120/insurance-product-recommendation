# 第二阶段成员分工总览

## 第二阶段目标

第二阶段不是重做系统，而是在第一阶段 MVP 的基础上补齐可演示、可追踪、可答辩的能力：

1. 训练记录和模型指标保存到 SQLite。
2. 支持逻辑回归、决策树、随机森林三类模型。
3. 提供模型指标对比 API 和页面图表。
4. 推荐结果保存到 SQLite，并支持历史查询和 CSV 导出。
5. 页面展示正负样本比例、模型指标对比、推荐等级分布。
6. 完成接口说明、测试记录、运行截图、答辩演示流程。

## 依赖顺序

```text
Task 1 SQLite 基础：已完成
Task 2 后端持久化：成员 A 先做
Task 3 模型对比：成员 B 基于 Task 2 做
Task 4 推荐导出：成员 C 可在 Task 2 后并行
Task 5 前端图表：成员 D 可先做页面容器，真实数据依赖 Task 3/4

Task 7 最终验收：组长统一完成
```

## 合并顺序

推荐 PR 合并顺序：

1. `phase2/backend-history`
2. `phase2/model-compare`
3. `phase2/recommend-export`
4. `phase2/charts-ui`
5. `phase2/docs-qa`

前端图表依赖后端接口和数据结果，文档最好最后合并，避免接口说明反复变动。

## 群通知模板

```text
各位同学，第二阶段开发从 phase2/integration 分支开始，不直接修改 main。

分工如下：
1. 后端持久化：训练记录、模型指标写入 SQLite，完成 /api/models/runs。
2. 数据与算法：新增决策树、随机森林，完成 /api/models/compare。
3. 推荐结果模块：推荐结果入库、历史查询、CSV 导出。
4. 前端可视化：完成数据比例图、模型指标对比图、推荐等级分布图。
5. 文档测试答辩：更新接口说明、开发看板、测试记录、运行截图、答辩演示流程。
6. 组长负责分支管理、PR 检查、全量测试和最终合并。

开发规则：
1. main 只放稳定版本，不直接提交 main。
2. 每个人从 phase2/integration 新建自己的 phase2 分支。
3. 每完成一个小功能就 commit。
4. 完成模块后发 Pull Request。
5. PR 必须写清楚完成内容、修改文件、测试命令、是否需要别人配合。
6. 每个 PR 至少运行 python -m pytest -q。
```
