# 组长集成验收开发文档

## 负责范围

组长负责第二阶段主分支、PR 检查、冲突处理、最终测试和演示版本发布。

## 工作分支

```text
codex/phase2-development
```

组长不在成员分支中直接开发功能，主要负责维护集成分支稳定。

## 工作地点

| 类型 | 路径 |
| --- | --- |
| 集成分支 | `codex/phase2-development` |
| 计划文档 | `docs/superpowers/plans/2026-06-05-phase2-development.md` |
| 分工文档 | `docs/phase2/` |
| 任务看板 | `docs/开发任务看板.md` |
| 接口文档 | `docs/接口说明.md` |
| 测试目录 | `tests/` |

## 每次合并 PR 前检查

```powershell
git fetch origin
git switch codex/phase2-development
git pull origin codex/phase2-development
python -m pytest -q
```

检查 PR 内容：

```powershell
git diff --stat origin/codex/phase2-development...成员分支名
git diff --name-only origin/codex/phase2-development...成员分支名
```

组长需要确认：

1. 成员只修改了自己负责范围内的文件。
2. 没有提交 `.venv/`、`__pycache__/`、数据库文件、模型文件、导出 CSV。
3. API 路径、字段名、页面路由没有随意变动。
4. 测试文件和功能代码一起提交。
5. PR 描述中有测试命令和测试结果。

## 最终验收命令

```powershell
python -m pytest -q
python -c "from app.main import app; print(app.title)"
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

浏览器检查：


## 最终演示流程

1. 启动系统。
2. 打开首页，说明系统目标。
3. 打开数据页，展示数据统计和正负样本比例。
4. 打开训练页，分别训练三种模型。
5. 打开评估页，展示模型指标对比。
6. 打开推荐页，生成推荐结果。
7. 导出推荐结果 CSV。
8. 打开 `/docs`，说明接口设计。
9. 展示测试记录和成员分工。

## 交付标准

| 项目 | 标准 |
| --- | --- |
| 自动化测试 | `python -m pytest -q` 通过 |
| 页面启动 | uvicorn 启动后 5 个页面可访问 |
| 数据读取 | `/api/datasets/profile` 返回训练集和评估集统计 |
| 模型训练 | 三种模型可训练并返回指标 |
| 模型评估 | 可展示最近模型和模型对比 |
| 推荐结果 | 可生成、查询、导出 |
| 文档材料 | 测试记录、截图、答辩流程完整 |
