## 工作目标

在逻辑回归基础上新增决策树和随机森林，并提供模型对比接口。第二阶段答辩时，需要展示不同模型在 Accuracy、Precision、Recall、F1、AUC 上的差异。

## 模型范围

| model_name | 模型 | 说明 |
| --- | --- | --- |
| `logistic_regression` | `LogisticRegression` | 第一阶段 baseline，保留 |
| `decision_tree` | `DecisionTreeClassifier` | 可解释性强 |
| `random_forest` | `RandomForestClassifier` | 集成模型，通常更稳 |

不引入 XGBoost、LightGBM、神经网络等额外依赖，避免安装复杂和答辩风险。

## 接口目标

| 接口 | 方法 | 目标 |
| --- | --- | --- |
| `/api/models/train` | POST | 支持三种 `model_name` |
| `/api/models/compare` | GET | 返回每种模型最近一次训练指标 |

`/api/models/compare` 返回格式建议：

```json
{
  "items": [
    {
      "model_name": "logistic_regression",
      "run_id": "xxx",
      "metrics": {
        "accuracy": 0.8,
        "precision": 0.7,
        "recall": 0.6,
        "f1": 0.65,
        "auc": 0.75
      }
    }
  ]
}
```

## 开发步骤

1. 在 `model_service.py` 中扩展 `SUPPORTED_MODELS`。
2. 修改 `_build_model(model_name)`，根据模型名称创建对应 sklearn 模型。
3. 保持训练输入、评估指标、保存模型 bundle 的字段一致。
4. 在 `history_service.py` 中增加“按模型取最近一次训练记录”的查询函数。
5. 在 `model_service.py` 中增加 `compare_model_runs()`。
6. 在 `app/routers/models.py` 中增加 `GET /api/models/compare`。
7. 测试三种模型训练都能返回 0-1 范围内的指标。
