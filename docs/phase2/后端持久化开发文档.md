# 后端持久化开发文档

## 负责人

成员 A：后端持久化。

## 工作目标

把训练记录和模型指标从进程内存升级为 SQLite 持久化记录。完成后，系统重启后仍可以查询最近训练记录。

```powershell
git switch -c phase2/backend-history origin/codex/phase2-development
```

| 文件区域 | 原因 |
| --- | --- |
| `app/templates/` | 前端可视化成员负责 |
| `app/routers/recommendations.py` | 推荐结果成员负责 |
| `docs/` | 文档成员负责，除非补充接口说明 |

## 接口目标

| 接口 | 方法 | 目标 |
| --- | --- | --- |
| `/api/models/train` | POST | 训练成功后写入 `model_runs` 和 `model_metrics` |
| `/api/models/runs` | GET | 从 SQLite 返回训练历史 |
| `/api/models/evaluate` | GET | 返回最近一次训练指标，缓存为空时可从 SQLite 读取 |

## 数据字段

训练记录字段：

```text
run_id
model_name
train_rows
eval_rows
model_path
created_at
```

指标字段：

```text
accuracy
precision
recall
f1
auc
```

## 开发步骤

1. 新增 `history_service.py`，封装保存训练记录、查询训练列表、查询最近训练记录。
2. 修改 `train_baseline_model()`，训练成功后生成 `run_id` 和 `created_at`。
3. 保存模型文件时，把 `run_id` 和 `created_at` 一起写入 joblib bundle。
4. 修改 `get_latest_run()`，内存缓存为空时从 SQLite 查询最近记录。
5. 修改 `list_model_runs()`，从 SQLite 返回列表。
6. 在 `app/main.py` 启动时调用 `create_tables()`。
7. 补充测试，测试必须使用临时 SQLite 数据库。
