# 首次启动会议说明

建议首次启动会控制在 30 到 45 分钟，目标是统一项目范围、技术路线、分工和交付节奏。

## 会议目标

- 明确第一版系统范围。
- 明确每位组员负责的模块。
- 明确 GitHub 协作方式。
- 明确第一次小交付时间。

## 建议流程

1. 说明项目目标：完成一个轻量 Web 保险产品推荐系统。
2. 说明技术路线：FastAPI + Jinja2 + SQLite + Pandas + scikit-learn + ECharts。
3. 说明核心流程：数据读取、模型训练、模型评估、推荐预测、页面展示、结果导出。
4. 分配角色：数据算法、后端接口、前端展示、文档测试、项目协调。
5. 说明 GitHub 规则：功能分支开发，Pull Request 合并到 `main`。
6. 确认第一阶段交付：最小可运行版本。
7. 确认每日同步格式和时间。

## 当天要交付的小成果

数据与算法同学：

```text
1. data.xlsx 和 eval.xlsx 的样本数、字段数。
2. 目标字段名称。
3. 正负样本比例。
4. 初步预处理方案。
5. 准备使用的模型和评估指标。
```

后端同学：

```text
1. 确认 FastAPI 项目可以启动。
2. 明确计划提供的页面路由。
3. 明确计划提供的 API。
4. 说明需要算法同学提供哪些函数。
```

前端同学：

```text
1. 首页布局想法。
2. 数据管理页面展示内容。
3. 模型训练和评估页面展示内容。
4. 推荐结果页面展示方式。
```

文档测试同学：

```text
1. 成员分工表。
2. 第一阶段开发计划。
3. 测试记录模板。
4. 答辩演示流程初稿。
```
